Analysiere jeden Regex-Block in provider.js nach ReDoS-Kriterien und sichere problematische Stellen ab.

**ReDoS-Kriterien (worauf achten):**
- Verschachtelte Quantifier: `(a+)+`, `(.+)*`, `(.*?)*`
- Exponentielle Backtracking-Muster: `^.*?` mit anschließendem optionalen Alternations-Muster
- Sehr langer alternating-Pattern mit `|` und `\s*\d+` auf jede Alternation

---

**Analyse der Pattern-Gruppen:**

**patterns[] und germanPatterns[] (Grundform: `/^.*?,\s*KEYWORD\s*\d+:\s*/i`):**
- `^.*?` ist lazy quantifier, matcht minimal — bei `replace()` kein exponentiales Backtracking da .replace() nur erste oder alle Vorkommen sucht und bei Missmatch schnell abbricht
- Muster ist safe: lineare Komplexität bei replace() da String von links nach rechts gescannt wird
- Keine Änderung nötig, aber Kommentar hinzufügen

**`/^.*?\s+-\s+\d+:\s*/i` und `/^.*?\s+\d+:\s*/i` (germanPatterns):**
- `^.*?\s+` kann mit nachfolgendem `\d+:` bei Strings ohne Match O(n) brauchen — akzeptabel
- Safe: kein verschachtelter Quantifier
- Keine Änderung nötig

**abridgedPatterns[] (Grundform: `/\s*\(KEYWORD\)\s*$/i`):**
- Literale Keywords mit `\s*` am Ende und Anfang — kein Backtracking-Problem
- Safe: keine Änderung nötig

**trailingRegex (Zeile 304) — KRITISCH (user-controlled input):**
```js
const trailingRegex = new RegExp(`^(.+?)[-,]\\s*${safeSeriesName}$`, 'i');
```
- `safeSeriesName` kommt von `escapeRegex(seriesName)` — special chars sind escaped ✓
- `^(.+?)[-,]` mit lazy quantifier: bei sehr langem seriesName ohne Match muss Engine alle Positionen prüfen — O(n*m)
- Risiko: seriesName selbst kann sehr lang sein (aus API-Response, nicht direkt user-controlled)
- Absicherung: Länge von seriesName begrenzen bevor regex gebaut wird:
```js
const safeName = this.escapeRegex(seriesName);
// Limit seriesName length to prevent O(n*m) in trailingRegex
if (safeName.length > 200) {
    // Series name too long for regex — skip trailing removal
} else {
    const trailingRegex = new RegExp(`^(.+?)[-,]\\s*${safeName}$`, 'i');
    const trailingMatch = title.match(trailingRegex);
    if (trailingMatch) {
        title = trailingMatch[1].trim();
    }
}
```

**Titel-Swap-Pattern (Zeile 331) — LANGER ALTERNATIONS-REGEX:**
Der bestehende monolithische Regex ist lang aber safe: Keine verschachtelten Quantifier, jede Alternative ist `KEYWORD\s*\d+` — linear.
Jedoch ist er schwer lesbar. Kommentar hinzufügen dass er bewusst geprüft und safe befunden wurde.

**subtitle split (Zeile 323) — lookbehind:**
```js
const parts = title.split(/:|(?<=\s)-(?=\s)/);
```
- Lookbehind `(?<=\s)` ist atomic — kein Backtracking
- Safe: keine Änderung nötig
